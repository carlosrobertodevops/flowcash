## 1. Stack Overview

- **Runtime / package manager**: Bun
- **Web framework**: Next.js 15.1.3 (App Router, `output: "standalone"`)
- **Language**: TypeScript (strict, path alias `@/*` → `./src/*`)
- **UI**: React 19, Tailwind CSS, ShadCN-style primitives, Recharts, lucide-react
- **API / mutations**: Next.js Server Actions (no separate Elysia layer in this snapshot)
- **ORM / DB**: Drizzle ORM + PostgreSQL 16
- **Validation**: Zod v4 (with `.superRefine` cross-field rules)
- **Auth**: `jose` JWT in HttpOnly cookie (`flowcash_session`, 7-day expiry); bcrypt (rounds 12)
- **Container**: Docker Compose (`app` + `db`)

## 6. Domain Rules (canonical)

- **Soft delete only** for accounts (`deleted = true`, `deletedAt = now()`); never physically delete user accounts. Hidden from user flows when `deleted = true`.
- **Status pairs**:
  - `payable` → `pending | paid`
  - `receivable` → `pending | received`
  Cross-field validation enforced in `accountInputSchema.superRefine`.
- **Recurrence**: monthly / yearly creates 6 future child accounts (1 parent + 6 = 7 rows total). Children share parent via `recurrenceParentId`, status `pending`.
- **Access control**: every query respects `tenantId`, role, owner, and `collaboratorEmails` (case-insensitive).
- **Categories**: empty category normalizes to `"Geral"`.
- **Currency**: stored as `numeric(12, 2)`; BRL ↔ USD via `EXCHANGE_RATE_BRL_USD`.
- **Session**: HttpOnly signed cookie (`flowcash_session`, JWT HS256, 7-day expiry). Verified on each action.
- **Passwords**: bcrypt hash only (rounds 12); plaintext never stored.
- **Quotas**: plans (`free`, `standard`, `business`) define `payableLimit` / `receivableLimit`; `0` means unlimited. Checked on create and on recurrence expansion.

## 7. Multi-Tenancy & Roles

- Every tenant-scoped table carries `tenantId` FK.
- Roles (`userRoleEnum`): `super-user`, `admin`, `standard`, `free`, plus legacy `user`.
  - `super-user` → manages all tenants and users.
  - `admin` → manages users within own tenant.
  - `standard` / `free` → see only own tenant accounts (subject to collaborator visibility).

## 8. CSV Import

- Validated by `importCsvSchema`.
- Splits rows by `type` before insert.

## 9. Conventions

- Named exports preferred; default exports reserved for pages/layouts.
- Arrow functions, early returns, ternary over nested `if`.
- Zod validation **on the server** before any DB write.
- `revalidatePath()` after mutations.
- Dark-mode via Tailwind `class` strategy.
- No custom hooks directory (built-in React hooks only).
- No TODO / FIXME flagged; no obvious dead code detected at scan time.

---

## 10. Oddities & Gaps

- `users.role` enum mixes legacy `"user"` with the current 4-role model (`super-user`, `admin`, `standard`, `free`).
- `createAdminUserAction` / `updateAdminUserAction` allow any of the 4 roles by schema; tighter enforcement lives in code, not the schema.
- Password reset tokens are stored in DB (hashed) — no ephemeral token service.
- No audit-log table; soft delete + `updatedAt` provide minimal history.
- CSV import logic continues past LOC 500 of `actions.ts` and was not fully read in this scan.
